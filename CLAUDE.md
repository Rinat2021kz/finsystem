# CLAUDE.md — брief проекта для Claude Code

Прочитай этот файл первым. Затем `SPEC.md` (полное ТЗ) и `schema.sql` (БД). Начинай строго с MVP (Фаза 1 ниже). Ничего из «отложенного» не делай, пока не готов MVP.

## Что мы строим
Управленческая финансовая система для малого бизнеса Казахстана (валюта — тенге). Клиент вносит простые операции (доход/расход/перевод), система сама считает ДДС, ОПУ, баланс денег, отчёты по проектам, планы и инвестиционные модели. Клиент НЕ видит формул — видит понятные отчёты и подсказки. Это же — рабочий кабинет финансового консультанта, который ведёт нескольких клиентов.

Полное описание, модули, формулы, роли, приёмочные тесты — в `SPEC.md`.

## Стек (согласован; менять только по явному запросу владельца)
- Next.js (App Router) + TypeScript — фронтенд и API в одном репозитории.
- PostgreSQL + Prisma (миграции строим на основе `schema.sql`).
- Auth.js: email/пароль или magic-link. Роли — в таблице `company_members`. Клиент без аккаунта — по `share_links` (токен в URL).
- Расчёты — серверные TS-модули в `lib/calc/` с юнит-тестами (Vitest).
- Экспорт: Excel (exceljs) и PDF.

## НЕНАРУШАЕМЫЕ бизнес-правила (ломают корректность)
1. **Деньги — только целые числа в тиынах** (1 ₸ = 100 тиын). Тип BIGINT / bigint. НИКОГДА не хранить и не считать деньги во float. Форматирование `3 750 000 ₸` — только на выводе.
2. **ДДС ≠ ОПУ.** `date_cashflow` — дата движения денег (ДДС). `period_pnl` — экономический месяц (ОПУ). Это разные поля, не смешивать.
3. **Перевод (transfer) — не доход и не расход.** Влияет только на остатки счетов. В сводном ДДС по всем счетам переводы не меняют общий остаток; в отчёте по одному счёту — учитываются.
4. **Никаких Excel-ошибок.** Любое деление защищено: если знаменатель 0 → возвращай `null` и показывай «нет данных»/0/предупреждение. Никогда не показывай ошибку пользователю.
5. **Закрытый месяц неизменяем.** После закрытия обычный пользователь не редактирует операции; только админ может открыть; всё пишется в `audit_log`; хранится снимок в `report_snapshots`.
6. **Мультитенантность.** КАЖДЫЙ запрос к данным фильтруется по `company_id`, и проверяется членство пользователя в `company_members`. Никогда не отдавай данные чужой компании.
7. **Отчёты — по выбранному периоду**, не по текущей дате. Никаких `TODAY()`-подобных зависимостей в логике расчётов.
8. **Расчётные величины не хранятся** в оперативных таблицах (маржа, долг, остаток, рентабельность считаются на лету). Исключение — `report_snapshots` для закрытых месяцев.

## Соглашения по коду
- TypeScript strict. Ясные имена. Комментарии на русском допустимы (доменные термины — ДДС/ОПУ).
- Все денежные типы — `bigint` (тиын). Утилиты форматирования/парсинга денег — в `lib/money.ts`.
- Даты месяцев храним как первое число месяца (DATE).
- Расчётные функции чистые (без side-effects), покрыты тестами из раздела «Приёмочные тесты» `SPEC.md`.
- Валидация входных данных на API (zod). Отрицательные суммы операций запрещены (знак задаёт тип).
